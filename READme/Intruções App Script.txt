## O que foi criado

**5 arquivos Apps Script** prontos para colar no Google Apps Script:

| Arquivo | O que faz |
|---------|-----------|
| [01_Setup.gs](nps-greenlife/apps-script/01_Setup.gs) | Roda 1x: cria as 9 abas, cabeçalhos, De_Para com ~85 mapeamentos, menu personalizado |
| [02_Importacao.gs](nps-greenlife/apps-script/02_Importacao.gs) | Importa Solvis e WeHelp mapeando colunas por nome (robusto a variações), consolida, aplica De_Para |
| [03_Utils.gs](nps-greenlife/apps-script/03_Utils.gs) | Config, log, triggers 06h/18h, menu "🏋️ NPS Greenlife" na planilha |
| [04_ClassificacaoIA.gs](nps-greenlife/apps-script/04_ClassificacaoIA.gs) | Gemini 1.5 Flash em lotes de 20, prompt v1.0, fallback automático, só classifica a partir de Abril/2026 |
| [05_WebApp.gs](nps-greenlife/apps-script/05_WebApp.gs) | Endpoint JSON com filtros por mês/região/unidade/plataforma para o dashboard consumir |

**Destaques técnicos:**
- O mapeamento de colunas usa busca por nome parcial — se a Solvis ou WeHelp mudar levemente o cabeçalho, o script continua funcionando
- Registros duplicados nunca são inseridos (controle por `id_unico`)
- Bootcamp é filtrado automaticamente em todos os pontos do pipeline
- Comentários de antes de Abril/2026 são importados mas não enviados à IA (economia de API)
- Se a IA falhar num lote, o script não para — classifica o lote com fallback e continua

**Próximo passo imediato:** seguir o [INSTRUCOES_INSTALACAO.md](nps-greenlife/apps-script/INSTRUCOES_INSTALACAO.md) — são 9 passos, o mais importante é rodar `setupCompleto()` e depois inserir a chave Gemini na aba Config.

Quando a planilha estiver funcionando e você confirmar que os dados estão sendo importados corretamente, iniciamos a **Fase 5 — Dashboard HTML**.
